# Drin Labs

Drin Labs is an open source security research organization building agentic systems that operate in the real world.

We design harnesses, backends, evaluation environments, and research tooling for one purpose: pushing autonomous security work past demos and into systems that can actually discover meaningful bugs in live software ecosystems. We care less about synthetic benchmark wins and more about whether a system can do useful work against real targets, under realistic constraints, with results that hold up to scrutiny.

That bias shapes how we release work here. We want tooling that earns its claims. In practice, that means our research is grounded in execution, and our bar for usefulness is simple: if a harness or workflow cannot help surface real issues in real software, it is not finished.

## What We Build

Drin sits at the intersection of:

- Agentic security research
- Autonomous bug-finding infrastructure
- Real-world agent evaluation
- Research tooling for high-signal security workflows

We are especially interested in the infrastructure layer around autonomous research: the environments, backends, orchestration systems, and testing loops that make serious agentic security work repeatable.

## Research Areas

A large part of our focus is offensive and adversarial research against open source applications, but the surrounding tooling surface is broader than that. Areas we care about include:

- Supply chain security tooling
- Defensive security systems for forensics, malware reverse engineering, and incident-oriented workflows
- Evaluation frameworks that measure agent performance in realistic scenarios
- Niche signatures and task-specific workflows built on top of existing tools
- Under-served security subfields where the tooling gap is still obvious

We are particularly interested in domains where experts already know the work is hard, the workflows are messy, and existing automation is thin.

## Philosophy

Drin has a strong preference for doers.

We value demonstrated capability over posture, and results over theory. If someone wants to contribute as a researcher or maintainer, the most convincing argument is not a pitch. It is evidence: shipped work, found bugs, strong ideas backed by implementation, and a clear ability to operate in difficult problem spaces.

We also think agent evaluation should be tied to reality. Releasing a benchmark, harness, or workflow is not enough by itself. The question is whether it can autonomously produce meaningful outcomes. That standard informs both the research we publish and the infrastructure we invest in.

## Open Source

Drin Labs is an open source lab. We publish research, release tooling, and welcome serious contributions from people who want to improve the state of agentic security work.

If you want to contribute to the software in this organization, open an issue or send a pull request.

If you want to work with us on research, contact [research@drin.dev](mailto:research@drin.dev). We are especially interested in researchers with a track record of using agents to find bugs and a willingness to share methods, lessons, and technical judgment.

If you have ideas for neglected areas of security tooling, contact [ideas@drin.dev](mailto:ideas@drin.dev) or open a discussion in this organization.

## Standards

We expect contributors to engage seriously, respectfully, and inclusively. Strong technical standards and open collaboration are compatible; both matter here.

This organization is intended to be useful to researchers, engineers, and maintainers who want to build credible systems and publish work that can stand on its own.
